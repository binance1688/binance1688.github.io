闪电网络生态

闪电网络是链下比特币交易的新系统，它允许用户在不需要像银行这样中央机构角色的情况下互相交易。

作为比特币的 L2 解决方案，可用于扩展微支付和日常交易，且通过使用智能合约和支付通道，两方可以以几乎零成本快速地进行比特币交易。

闪电网络（Lightning Network）由Joseph Poon和Tadge Dryja在2015年首先提出，被认为是比特币创立以来最重要的革新。

它利用了比特币的安全特性，在线下提供高速的实时交易处理能力。用户既可通过点对点的直接支付方式，也可以通过网络路由的方式实现间接支付。

闪电网络并没有发明新的加密形式，也没有使用新奇的比特币脚本，却巧妙地实现了离线支付的功能。以下为部分闪电网络应用生态。

TaprootAssets(Taro)

这是由Lightning Lab在10月18号推出的资产协议，为什么要单独介绍这个协议呢？因为有了这个协议可以在闪电网络上面创建资产，然后通过闪电⽹络发送，以最⼩的成本进⾏快速，⼤量的交易。比如发⾏同质化的资产（稳定币），也可以发⾏NFT。下面是TARO的生态项目：

Nostr

NostrNostr 是一个分布式社交媒体协议，旨在创建一个抗审查的全球社交网络。该协议不依赖于受信任的中央服务器；相反，所有用户都运行一个客户端。使用此客户端，用户可以通过撰写帖子、使用他们的私钥对其进行签名并将其发送到其他服务器来发布内容，然后由其他服务器转发该内容。

Twitter 创始人 Jack Dorsey 向去中心化社交网络 Nostr 捐赠约 14.17 枚 BTC（约合 245,000 美元），以进一步资助 Nostr 的开发。

Primal

基于 Nostr 协议的社交应用，其社交应用程序套件可以将类似推特的界面与开源 Nostr 协议相结合，并且优先考虑审查、帐户所有权和去中心化等功能。2023-07-12，Primal 完成 100 万美元融资。

Nostr Assets Protocol  

Nostr Assets Protocol 是一个开源协议，它将 Taproot 资产和 Satoshis（比特币单位）引入 Nostr 生态系统。一旦引入资产，用户就可以使用 Nostr 的公钥和私钥在 Nostr 协议层发送和接收它们。资产的结算和安全仍然依赖于闪电网络，而 Nostr Assets Protocol 本身并不发行资产，只是通过协议将资产引入 Nostr。

ORDINALS（当前最火热的BTC生态）

Ordinals在今年五月份的时候，我们发过文章跟大家介绍，这是一个整体的概念。Ordinals创始人是者 Casey Rodarmo， satoshis 的编号方案，允许跟踪和传输单个 sats。这些数字称为Ordinals。

Satoshis 按照它们被开采的顺序编号，并从交易输入转移到交易输出，先进先出。编号方案和转账方案都依赖于 顺序，编号方案依赖于挖矿的顺序，而转账方案依赖于交易输入和输出的顺序。

在 Ordinals 协议发布 2 个月后，推特用户 @domodata 在 Ordinals 协议上，提出了一个比特币上的 Token 标准 — — BRC-20 标准。这就是BRC20的由来，而万倍神话的ORDI是由domo作为实验推出的首个代币。初期铸造成本1-3U。

目前ORDINALS下面主要是两个应用，一个是NFT，一个是铭文即BRC20。

虽然铭文赛道这一块不止是BRC20,还有很多BRC100,ORC20,SRC20等等，经过半年的市场验证，我们现在可以看到建设在ordinals上面的铭文赛道除了BRC20，其他的几乎都没发展起来。

BRC20目前来看，ordi已经完成上所目标，SATS下个目标是B安，而两者的市值来说未来可以进前五十名，至于的动物园风险很大，因为五月份那批meme代币几乎没有起来的，这一轮流行的都是有社区支持的新币种。

说完铭文赛道的BRC20，我们再看小图片NFT，这也是ordinals重要的组成部分，不管是以太链上的NFT还是BTC链上的NFT，都是比直接买卖代币难度要大，对于普通玩家的门槛比较高，毕竟很多人难以理解一张图片的价值。而早期的事物你就算再难以理解我们也建议参与一下，无非就是使用新工具去进行操作，并没有太难。

我们始终认为ORDINALS 小图片目前还是价值洼地，未来会出现比肩BAYC以及PUNKs的NFT。

BITVEM

它是“⽐特币虚拟机 Bitcoin Virtual Machine ”的缩写。在10月9号由robin_linus提出，通过比特币进行激励的去中心化文件托管服务 BitStream，无需依赖信任及密码学，客户端和服务器使用 optimistic 协议执行文件 coin 原子交换。目前暂时没有应用，但是热度很高，是继ORDINALS之后的比较被大家关注的协议。
