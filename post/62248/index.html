作者：橙子Orange

最近看到一句话：Friend.tech 是 Web3 社交赛道狂暴牛市的开端，Web3 SocialFi Summer 很可能就会在意想不到的时刻发生！ 我不否认 Friend.Tech 确实出彩，通过绑定与量化粉丝与 KOL 的利益关系快速在市场上筑起旁氏高楼，给玩家带来巨大财富效应，但这终归结底还是一场依赖经济模型塑造的多巴胺金融游戏，并未从本质上真正带动 SocialFi 的发展。而 SocialFi 的繁荣一定需要来一场由下而上的变革，社交产品的底层架构中数据也一直以来是一个绕不开的话题，本文聚焦 SocialFi 发展的瓶颈本质与数据问题的关联，并从社交产品使用去中心化存储的重要性与意义展开讨论。

SocialFi 的瓶颈在哪？

我之前就在某家做 SocialFi 产品的 Web3 公司实习，公司具体产品逻辑有点类似于 Friend.Tech，也是围绕 KOL 与粉丝经济有关，所以关于本段标题这个问题，我特意还与前老板通宵达旦的探讨了一番，探讨到凌晨 4 点最终得出一个结论，现在的 SocialFi 应用还是太 Web2 思维了 ，但这绝对不是产品经理和创始人的错，是因为 SocialFi 定义和精确概念是什么没有人能说清楚，因为这个命题太庞大了，很多 Web3 社交应用同属于 SocialFi 赛道，但产品逻辑和经济模型天差地别，况且行业也完全没有对于 SocialFi 的共识，所以大家只能从 Web2 中去抄，然后再加上一些旁氏的经济模型，这就是 Web3 SocialFi 的现状。

我的前老板有一段话我特别深刻，当晚我脑海中一直思考这段话彻夜难眠，原话如下：

“大部分的 SocialFi 项目是为了 SocialFi 而 SocialFi，很多应用注重 Fi 固然没错，因为确实 Web3 不需要纯 Social 的产品，但都没有领悟 Social+Fi 结合的奥义，虽然我也不知道到底怎么结合，但这个结合不是单靠旁氏+’链改‘来解决的。”

是啊，Web2 中像抖音（Tiktok）、微信等已经发展起来的社交项目都是基于高频、大规模与刚需，那么明显目前 Web3 的效率和用户人数都拼不过 Web2，而 Web3 SocialFi 应用想要超越 Web2 已有的应用一定是要从人性角度思考的，区块链的 Token 只是一个经济存在形式，并不是 Web3 社交产品最本质的东西 ，而是要利用区块链来解决人类某些刚需问题，比如生存、隐私保护、信仰等等。

Web3 SocialFi 应用单纯通过代币激励来吸引用户的叙事已经被证伪， 因为即使成为爆款当红炸子鸡也终会有死亡螺旋的那一天，并不能发挥出真正的「Social+Fi」的组合拳。SocialFi 的应用层之下，也正需要有一个基础设施层去进行一场从底层架构开始的变革，而变革也需要从数据问题开始。

为什么 SocialFi 应用需要去中心化存储？

SocialFi 最重要的意义之一就是让用户实现真正对自我数据的控制权，这是社交应用数据产生经济价值的前提与保障。 目前的 SocialFi 产品如果要实现真正意义的 Web3 社交产品，那么必须关注其底层的数据存储，因为数据存储的方式决定了 SocialFi 产品的用户上限，也决定了一款 SocialFi 产品到底能不能满足用户在 Web3 时代的社交愿景。那么 Web3 时代真正一款优秀的 SocialFi 产品应具备哪些特点呢？

1. 数据永久存储： 社交应用程序需要存储用户所产生的内容，如帖子、图片、视频和其他数据。而传统的社交媒体平台通常使用中心化的服务器来存储这些数据，一旦平台背后的公司崩盘或破产，用户将失去一切数据。社交应用只有采用 Arweave 这种去中心化存储解决方案，不仅可将用户数据分布式存储在区块链网络上的多个节点中，还可以实现永久存储，这有助于提高数据的可用性。

2. 无视任何中心化组织的监管： Web3 本身就是一场崇尚自由的运动，而 Web3 社交产品更不能剥夺每个人说话的权利，也应减少或甚至消除权力机构对内容的审查。SocialFi 产品应采用 DAO 等去中心化的治理方式，社区成员共同决定平台规则和内容审核标准，通过借鉴或合作例如 PermaDAO 等一些具有成熟的 DAO 范式的组织，可以最快的速度实现真正的社区治理。

3. 去中心化社交身份： 在传统社交媒体平台上，用户的身份通常由平台所定义，并且与平台紧密绑定。在 Web3 时代，SocialFi 产品应鼓励去中心化的社交身份，让用户自己管理和控制他们的社交标识。这也意味着用户可以跨不同社交平台使用相同的身份，并且不会受到单一平台的限制。

4. 用户数据隐私保护： SocialFi 产品中用户的数据隐私一定是至关重要的，这意味着用户拥有对自己的数据完全控制权，可以选择与谁分享，以及分享什么样的信息，用户也可以匿名参与社交，并避免第三方跟踪或滥用他们的数据。现在有很多应用也已经集成由 Arweave SCP Ventures 与 Binance Labs 共同投资的 Mind Network 其开箱即用的 SDK 作为动态数据层来实现隐私保护。

5. 完善的创作者经济机制： Web3 的经济模型将鼓励用户积极参与社交平台。通过代币、NFT（非同质化代币）奖励和治理机制，用户可以分享自己的内容、参与社交互动，并为社交生态系统的健康做出贡献。用户不再仅仅是被动的数据提供者，而是成为社交网络的主人公， 这将激发更多的用户创造内容和参与社交网络。

6. 可扩展性与跨链互操作性： 一款能推广至世界的 SocialFi 产品应实现不同平台甚至区块链之间的互操作性，也需要提供数据可扩展性来满足用户能够跨不同的社交网络共享内容和互动的需求，借助基于 SCP 范式来建设社交应用可以根据用户数据的实际需要添加更多的存储节点，目前诸多链上应用采用模块化的 PermaWeb 标准服务的 API 接口来实现产品无限组合性、用户自由度和社交数据的可移植性，这有助于确保 SocialFi 产品的用户上限可以不断扩大。

抽丝剥茧，这些以上几点愿景的本质其实就是应致力于把一切权力还给用户，而实现路径中最重要的就是数据应采用去中心化存储， 去中心化存储可以赋予用户更多的控制权，他们可以更好地管理自己的数据，并决定是否分享或销毁数据，在确保用户的个人信息和通信内容得到充分保护的情况下能以最大限度提高用户对于数据的自由度。

值得欣慰的是，目前有很多耳熟能详的 SocialFi 应用，例如 Mirror、Lens 等应用的出现也给 Web3 创作者经济、去中心化社交图谱等细分领域设立了标杆，尤其 Mirror 作为 Web3 内容发布平台，用户在 Mirror 中发布的文章会存储在 Arweave 区块链网络中，Arweave 也是目前唯一支持数据去中心化且永久存储的解决方案（至少存储 200 年），目前也有很多项目正在积极拥抱数据永久存储的解决方案，因为 Arweave 其提供的稳定性与永久性价值不是简单用金钱衡量的。 同时 Mirror 用户也完全可以掌控自己的内容，不受任何中心化组织的限制与审查。

SocialFi 一定是基于有效社交展开的，有效社交也一定是有意义的，哪怕只是满足很小众的需求。更多 SocialFi 产品应该去思考社交+金融深度结合的意义与本质，同时也应重视起去中心化永久存储对于社交数据的重要性与必要性，我们也期待看到 Web3 社交产品能真正满足用户刚需的史诗级应用出现，也期待着 Web3 SocialFi Summer 到来的那一天！
