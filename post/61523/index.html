原文作者：David Hoffman

原文编译：深潮 TechFlow

以太坊现在已经八年了，而我已经陪伴了其中六年。

我们现在知道去中心化的加密网络适用于什么，以及如何扩展它们。我们在当前对加密网络的理解中还有许多宝贵的收获。

然而，加密的终极目标是什么仍然有很多未解之谜。尽管迷雾正在逐渐消散，但它仍然存在。

对于在这个领域剩下的一些重大问题，无论是个人还是在我的风险投资职责中做资本分配决策，都依赖于这些问题的答案，其中有许多未解之谜。加密领域的所有风险投资家和开发者都在试图更准确和更快地回答这些重大问题，超过他们的竞争对手。

我花了一些时间来确定该领域留下的一些最大问题以及我对它们的看法。

#1 ：许多超级链还是单个超级链？

我们知道以太坊将如何扩展。

Rollups 已经将以太坊的区块空间扩展到了丰富的 L2 区块空间。在 2020 年和 2021 年，我们在理论上讨论过这个问题，而在 2022 年和 2023 年，我们看到它开始投入生产。

但是有很多团队以他们独特的方式执行着同样的愿景。Optimism 超级链！Arbitrum Orbits! zkSync 的 ZK Stack！Polygon Supernets! Eclipse! 有太多不同的方法来构建 L2！

每一种方法都代表着一种战略，将以太坊 L1 的区块空间扩展到互联网的最远角落。我对加密的终极目标是，一条区块链可以找到它的位置在互联网的每个角落，而以太坊正在生成各种不同的区块链，每个区块链都有其独特的专长，来填补互联网的空白。

但问题仍然存在：

  • 我们是否需要那么多不同的 Rollup 标准？或者只需要一个框架就够了吗？我的区块链帝国模型是否自然扩展到“以太坊 L2 上的帝国模型”？

  • 或者以太坊的 Rollup 中心路线图是否从根本上降低了其他 Rollup 标准的进入门槛，产生了更多的 Rollup 战略的多元平衡？

  • 应用程序特定的 Rollup（roll—apps）能否在经济上证明自己的合理性？

  • 我们今天可以预测这个理论上的赢家将具备哪些特性？

  • 还是经济学会迫使所有应用程序汇集到少数主导的 Rollup 上？

  • 经济和可组合性激励是否会迫使走向单一的庞大以太坊 Rollup，正如 Vitalik Endgame 中的其中一种结果所示？

更少的 Rollups？

Rollups 有成本，只有一些用例能够仅基于经济因素来证明建立和维护 Rollup 网络的成本是合理的。这些应用程序将不得不找到一个更一般化的领域，一些一般化的 Rollup 将竞争提供最便宜的地产。

这个观点还涵盖了一些可组合性的好势头。当更多应用程序在同一链上运行时，选项增多。整体大于部分的总和，部分越多，整体越大。人们自然而然地迁往城市，而 Rollup 经济学也是如此。谁能建造最大的城市？

更多的 Rollups？

随着技术的发展，成本会降低！随着技术的成熟，Rollup 部署的固定成本会随时间减少。

虽然不断发展的 Rollup 似乎难以忍受，但研究和创新的其他领域可以帮助控制这一点。同质化的区块空间，抽象层，跨链执行，合同调用，共享排序和离链意图都将有助于终结 10, 000 条链的混乱。

在一个链上有每个应用程序共存是令人钦佩的，但对于单一链来说，存在着规模的基本限制。虽然通过多个 Rollup 的水平扩展似乎混乱，但在多个方面还有许多研究和开发的机会。

多元化的路径？

以太坊超级链的未来是令人信服的，但它并没有完全解决 L2 的可组合性问题。

以太坊的 Rollup 中心路线图提供了一条路径，通过简单地允许根据需要部署链来实现无限扩展。如果一个 L2 变得拥挤，只需启动另一个即可！但这种策略为以太坊提出了 Solana 社区非常乐意告诉你的新问题 —— 主要是可组合性问题。

像 OP Stack 这样的 Rollup SDK 有助于解决这个问题。共享的标准和同质化的区块空间是重新组合不同链回到一个单一的执行层的一个巨大的第一步，但这次是无限规模的。增加一些共享执行，跨链合同调用，一些 UI 抽象，我们的 L2 超级链拥有了无限的规模。

只是有一个问题。

Optimism、Arbitrum、Polygon 和 zkSync 都希望这样做。

前进的道路

如果 Optimism 超级链被认为是 1, 000 条不同的链，那就太好了，但 Arbitrum 仍然使用一种不同的语言作为 Optimism，就像所有其他 L2 SDK 一样。

这就是为什么 Arbitrum 的超级链版本看起来不像 Optimism 的版本。Arbitrum 对单一统一的超级链愿景感兴趣，其中通用的可组合性机制将任何和每一个链绑在一起。

Arbitrum 并不是在制造另一个超级链，而是在这些链之间的空间工作。这个关注点是以太坊 L1 和超级链结算之间的互操作性层。一旦超级链就其内部状态达成一致，Arbitrum 希望专注于在集体的最终性通过发布到 L1 之前的链间结算的研发努力。

这是一个令人信服的愿景，如果你仔细观察，你会看到上面的两个部分实际上是一样的。可能有一个未来，其中存在许多 rollups，链可组合性技术的创新允许它们混合在一起，模糊成为一个单一的超级链。或者…这些技术不起作用，达到真正无缝的可组合性的唯一方法就是只有一个单一的 Rollup。

所以，以太坊的未来是…

几个不同的多链经济区？（超级链)

一个可组合的超级链？

一个单一的整体 Rollup？

#2: 在 Rollup 堆栈中，价值是如何被捕获的？

每个 L2 团队都希望在加密领域中传播他们的链开发套件。有 OP Stack、ZK Stack、Supernets、Orbits 等，肯定还有更多的跟进。

他们为什么要制作这些东西？L2s 从他们链的复制部署中有什么好处？由于分叉是无需许可的，当分叉是免费的时候，L2 代币如何捕获价值？分叉的过程中将代币分叉出去。为什么一个被分叉的 OP——Stack 链愿意向 OP Collective 支付费用？有什么激励？

Mantle，最大的 OP Stack 分叉之一，分叉了一个较旧版本的代码库，并表示他们目前没有计划加入即将到来的 Optimism 超级链。他们打算为自己保留自己的排序费用。

与此同时，Base 将其 15% 的排序费用贡献给 Optimism Collective，有效地变成了 OP 代币。所以，作为 OP——Stack 的一个分叉，Base 为 OP 贡献了价值，但 Mantle 没有。

这是什么原因？这两条链之间有什么区别？答案是：治理。没有治理，我们有不同的、混乱的、杂乱的链。有了治理，我们有同质化的区块空间、代码重用和共享的可升级性。有了这些属性，我们就有了开始融合 10, 000 多条杂乱链成为一个无链用户体验的基础。

这就是为什么我对 Optimism 的策略和路线图比其他所有的都感到特别被吸引的原因。Ben、Jing、Karl 和 Optimism 团队跑过了以太坊扩展的想法迷宫，并在任何其他团队之前得出了治理的逻辑结论，并从第一天开始一直在走困难的路，搞清楚去中心化的治理。所有其他的 rollups 都在技术优势上竞争，但最终，那将耗尽，他们将不得不开始搞清楚他们的长期治理策略，一旦他们得出那个逻辑结论。

与此同时，OP 堆栈可以吸收其他人开发的最好的技术，同时努力挖掘一个不可分叉的治理护城河。

“为什么治理是 rollup 竞争的逻辑结论”是一个超出这篇文章范围的宏大主题。这必须非常深入的研究 Optimism 的兔子洞，让你自己弄清楚。

然而，我对此还有以下疑问：

L2 在吸收周边技术时，像 OP 堆栈这样的通用、模块化框架是否能够吸收最好的 L2 技术，就像我们看到的以太坊 L1 那样？

加入超级链的激励力度有多大？这个问题是衡量“治理有效性”的代理；L2 治理能够变得多么有效？它是否足以将一堆杂乱的链连结在一起？

如果跨链可组合性创新不产生所需的创造无链体验的效果，治理可以利用哪些其他因素来增加 L2 代币价值捕获？

那 Rollup 基础设施提供商呢？

治理只是 L2 价值捕获方程的一半。即使 L2 SDK 能够捕获代币价值，他们仍然必须应对“其商业模式中 RaaS 大小的漏洞”。

如果我们认为将会有很多的 L2 rollups，那么这意味着将需要有基础设施来托管所有这些 rollups。

这就是像 Conduit 和 Caldera 这样的公司出现的原因。他们希望托管尽可能多的 rollups，以捕获 rollups 产生的部分费用。

像 Conduit 这样的 Rollups 作为一种服务与像 OP——Stack 这样的 L2 SDKs 进行了拇指大战。RaaSs 希望收费，而 L2s 也希望收费。平衡点在哪里？

我看到的有两种结果：

RaaS 供应商希望获得所有费用，并将试图绕过 L2 团队来阻止他们。

RaaS 供应商将接受他们受制于 L2 团队，并将简单地接受他们被给予的任何费用。

我在这里有 L2 偏见，所以这种推理可能需要被检查，但这就是我看到的。假设 RaaS 供应商是最贪婪的，并希望要#1 中所提到的东西。

RaaS 供应商：“我们拥有所有基础设施；当软件可以自由分叉时，为什么我们需要向软件纳税？”所以，RaaS 供应商只是取得 OP Stack 并帮助团队使用他们的 RaaS 部署 OP Stack 链，并且他们收集他们运营的所有链的排序费用，而 L2 代币得不到任何东西。他们甚至可以通过帮助他们的所有链共享排序来解锁一些链可组合性的好处。

这里的问题是，我们回到了治理问题。生产大量的链并不是一个足够的结果，虽然有一个单一的 RaaS 供应商解锁了一些共享排序的好处，但这远远不足以产生一个成功的超级链所需要的无链结果。如果一个 RaaS 希望尝试在 L2 SDKs 上赢得大战，他们…需要成为他们自己的 L2 SDK。这意味着他们正在进入 L2 竞争的舞台，他们最终会发现需要在 L2 技术创新、业务发展和治理等方面开展工作。

RaaSs 如何实际上去中心化他们的 L2 技术堆栈，如果他们的重要竞争优势是他们在物理位置运行物理硬件？如果任何 RaaS 通过运行每一个 L2 链产生了一个垄断，所有的 L2s 都集中到那一个 RaaS 托管中心。为了去中心化，RaaSs 将需要研究所有其他 L2s 已经工作了多年的同样的事情。

所以，这将他们推入#2 。RaaS 将成为 L2 团队的服务提供商，并将不得不与其他有竞争力的 RaaSs 竞争费用，因为 L2 团队使用他们的反垄断权杖确保没有一个单一的 RaaS 产生一个垄断，给予 RaaS 对 L2 SDK 过多的权力。

#3: LST 的平衡点在哪里？

在这里，我有些矛盾。我理解市场力量将迫使单一流动 LST 趋同的令人信服的论点。我并不天真地忽视这些因素。但我也不是那么悲观，以至于我愿意“轻易地妥协”，当面对与以太坊核心社区坚定持有的价值观和信仰相抵触的力量时。这里并不只是市场力量在起作用。

此外，一个 LST 的主导地位与次级 LST 进行吸血鬼式攻击的动机是相关的。当一个 LST 超越其基础时，吸血鬼攻击的欲望和力量相应地增加。这至少在短期内，可以打破主导 LST 的平衡。

关键的问题是：

以太坊社区愿意走多远来确保 LST 的多样性？

抑制单一 LST 优势的工具有多有效？

如果（何时？）一个单一的 LST 实现了完全的垄断，这会多大程度地侵蚀以太坊的核心价值观？

对我来说，多元主义的价值深深地吸引了我进入这个领域，我希望在任何可能的地方都能看到更多的这种价值。

如果我们最终确实汇集成为一个主导的 LST，那么到达那里需要多长时间？慢一点是不是更好？在我看来，答案是肯定的。在我们还有时间的时候，我们可以构建哪些防护栏？

我们最近看到，对于在 Arbitrum 上向 stETH 提供的 ARB 代币激励的投票被否决，主要是因为对 Lido 在以太坊上的主导地位的担忧。如果这里只有“市场力量”在起作用，那么这次投票应该是通过的。

#4: Solana 会被并吞吗？

Solana 和以太坊是否会作为独立的生态系统发展，还是它们之间的界限会融合？如果它们融合，以太坊对 Solana 有多大的引力？

没有人能够有效地反驳我对区块链的帝国模型，还有一堆类似的论文（Fat Protocol、L1s 是货币）都与它相一致。L1s 正在为完全的主导地位进行激烈的竞争，随着时间的推移，一个区块链最终会吞并所有其他的区块链。这是开源系统的本性，特别是当你加入经济激励的火箭燃料时。

Solana 在我看来与其竞争对手有所不同。它不是 EVM 的 L1 分叉，其中它创造的任何价值最终都会回流到以太坊生态系统。它不是 Cosmos，没有实际的结算层或神圣的 L1 货币。它不是比特币，所有的非 BTC 价值都被剥离和删除。

Solana 拥有它的虚拟机、扩展策略和 L1 资产：SOL。Solana 的整个技术堆栈都不是以太坊，这使它尽可能远离以太坊的引力。这种策略对我来说很有意义，因为以太坊似乎吞噬了其领域内的一切。对于任何非以太坊 L1 来说，最好的生存机会就是远离以太坊的影响。

然而，Solana 并不是生活在一个真空中。Eclipse 正在将 Solana 的虚拟机移植到以太坊上，将 Solana 的执行结算在更大的结算网络上：以太坊。

我认为 Eclipse 是“Solana 技术对 SOL 价值的背叛”，并选择加入以太坊的货币网络和结算层。Eclipse 实现了 ETH—maxi，即所有好的技术最终都会找到它的方式来到以太坊，特别是当它只是一个执行层，一个可以从少数结算层脱离，加入更全球化的层时。

那么，未来会发生什么？Solana 能保持其边界吗？

或者，更真实的是，无论一个 L1 距离以太坊的引力有多远，最终以太坊都会吞噬你，你从少数派网络转向多数派网络的时间越早，你的处境就越好吗？投奔以太坊的动力有多强？

#5: 我们如何实现链上价格发现？

我们在 Bankless Ventures 关注的一些最令人兴奋的交易围绕着这个问题。在链上进行价格发现，而不是通过 Binance，将为整个行业带来巨大的推动力。价格发现代表了去中心化和中心化系统之间的权力平衡，到目前为止，价格发现是中心化阵营牢牢握着的奖杯。

如果去中心化系统要“胜出”，我们需要那个奖杯。加密经济系统是真理机器，但现在，关于加密价格的真相并不是来源于托管资产的系统。我们需要在这里实现完全的闭环。加密货币产生了资产，并需要成为它们价格的预言机。

至少，这就是我们需要达到的地方。有各种有希望的机制可以帮助我们倾向于去中心化系统，但不清楚这会带我们走多远。Binance 拥有 1 毫秒的出块时间优势。没有任何去中心化系统将能够跟上这个速度，价格发现自然会趋于最有流动性和更新速度最快的预言机。

我们如何在链上实现这个？

Uniswap Hooks 和意图领域中蕴含着有望打破力量平衡的创新。意图可能是这里的关键。有一个价格发现的领域发生在 CEX 和 DEX 之间的空间。这个模糊、未定义的空间是做市商和 MEV 机器人做出决策和执行交易的地方，而不是在任何特定场所。以太坊的空间港口需要适应这个存在的层面。我们需要建立支持在链上 DEXs 和在链与 CEXs 之间的空间里穿梭的无尽种类的飞船之间的交易的基础设施，这样我们就可以鼓励它们更接近我们，远离 Binance。

只要我们的加密系统生活在 CEX 价格发现的阴影下，我们将永远是自己的劣质版本。实现链上价格发现将是我们行业成熟和复杂性的最重要的信号之一。

这是没有商量余地的；我们需要这个。如果没有链上价格发现，加密实验在某种重大方式上已经失败。

产生链上价格发现的机制肯定会是加密货币最有价值的基础设施之一。它也不一定是单一的灵丹妙药！Uniswap 和 AMM 是加密货币工具带中用于产生链上价格发现的巨大资产，但需要更多类似的机制。

谁将建造它，它看起来像什么？

结论

这篇文章涉及的是问题，而不是答案。

2023 年，人们还会提出许多关于加密货币的其他“问题”；这些只是我首先想到的五个大问题。

在其以 rollup 为中心的路线图的背景下，以太坊的可组合性和链抽象是如何工作的？

在以太坊的以 rollup 为中心的路线图中，价值如何确定？

LST 对以太坊的威胁有多大，它的未来有什么展望？
