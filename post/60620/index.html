2017 年至 2022 年左右的 NFT 与 1990 年代早期的网站类似。

早期的网站通常是由编程专业知识有限的人（比如我自己）使用基本的 HTML 和 CSS 构建的。它们往往是静态的，难以更新或扩展，并且是为单一特定目的而设计的。

1989 年 Web（Web 是“World Wide Web”的简称，也叫万维网，是一种建立在互联网上的网络服务，为浏览者在互联网上查找和浏览信息提供图形化的、易于访问的直观界面，其中的文档及超级链接将互联网上的信息节点组织成一个互为关联的网状结构）的推出标志着“超链接”时代的开始，通过 1991 年第一个网站的推出，实现了轻松导航、重塑内容结构和民主化共享知识。

在 NFT 的下一阶段，它们将越来越类似于现在的应用程序（App）或网站，并最终成为自主代理（Autonomous agents），提供增强的功能和更好的用户体验！

假设 NFT 将走向更加动态化、更加有状态的未来，我们可以将 NFT 的链上堆栈分解为与现在网站类似的结构 ：

已经有大量的标准和协议在 NFT“前端”领域运行，比如：ERC 4906、7496、721M、721TL 等。主要致力于修改艺术作品、元数据描述、特征和视觉内容。相关项目比如：Prrf beauty、Mathcastles、area.tech等。

然而，在 NFT“后端”领域，过去 6 年（2017 年至今）并没有大量创新。所有状态逻辑要么位于自定义合约上，要么位于中心化数据库上，通过中心化数据库为游戏玩法以及其他场景索引链上事件。糟糕的是，这种方式根本没有任何 utility 可言。

得益于 ERC-6551，有史以来第一次，我们拥有了一个标准化的 NFT 链上“后端” 。

定制化合约，包括我们为加密猫 和 CHZWZRDS 撰写的确实很棒，但它们本质上毫无不可组合性。

TBA 是 NFT 的通用账户/utility 层。

这一切与超链接和 1990 年代的网站有什么关系？

NFT 和网站都将有类似的繁荣期 。

没有人知道 NFT/网站的功能或用途，但每个人都知道它将改变一切。

如果大家在互联网泡沫破灭后放弃了网站/互联网，那么我们就会错过拥有长期来看真正重要的、更有趣的应用/产品的机会，比如：ebay、google、facebook、uber、airbnb 等。

过去有时可以预示未来。

处于婴儿期的 NFT 仍然具有巨大的增长潜力，但它必须从一开始就加倍努力使其与众不同：

- 数字唯一性

- 可拥有的物品

- 可验证的历史

图片来源：https://www.youtube.com/watch?v=bERVm10ryhY

我们即将见证“超链接”时代（或者如 @js_horne 所说的超媒体/加密媒体）的爆炸式增长。

NFT 开始去链接其他链上代币，并创建我们所说的所有权和关系的代币图谱（token graph），从而实现“链上”超链接。

超链接彻底增强了网站，就像维基百科（wikipedia）之于信息高速公路。它们通过连接相关内容来催生上下文。将“NFT”替换为“网站”，其意义就显而易见了。

NFT 自然会开始通过以下方式链接到其他链上对象/代币：

ERC-6551 为我们共同的链上历史赋予更多意义、深度和叙事， 就像超链接如何为 Web 提供重要的上下文信息一样。

几年后，你将不再称 NFT 为“价格虚高的 JPEG”。它们将成为由区块链驱动的互联网原生范例。创造不断丰富的梦幻般的体验，从一个代币到另一个代币无缝流动。更重要的是，那才是乐趣开始的时候！
