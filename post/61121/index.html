OP Crypto 团队在大会之前对 TON 就产生了极大兴趣，结合一些过去的研究与最近的更新，让我们在本文探讨 Telegram 和 TON 结合的无限可能性以及潜在的投资机会。

What is Telegram?

Telegram 是一个基于云的免费即时通讯软件， 2013 年由 VK（俄罗斯主流社交媒体之一）创始人 Nikolai 和 Pavel Durov 两兄弟创立。目前 Telegram 服务器遍布全球，运营中心和开发团队均位于阿联酋的迪拜。   

Telegram 拥有巨大且面向全球的用户群体，近几年的增长极快。据悉至今，Telegram 的月活用户（MAU）已接近 9 亿，注册用户超过 13 亿。

非常有趣的是，Telegram 是 Crypto 的“重要阵地”。订阅数/会员数最多的 Channel 以及 Group，超过一半是与 Crypto 内容相关的。几乎所有 Web 3.0/Crypto 项目都拥有 Telegram 社群，以便于实现其社区交流、项目更新、宣发等目的。

另外比较有趣的是，有许多跨境电商卖家通过公域（比如 Tiktok）引流到 Telegram 成交，Telagram 还是一个电商消费交易场所。

What is TON？

TON 是 2018 年由 Telegram 发起的项目，但随后遭到被美国 SEC 指控非法募资，所以在 20 年 5 月，创始人 Pavel Durov 宣布结束 Telegram 参与区块链开发。但在 Telegram 放弃项目后，被社区接受继续开发，项目由原来的“Telegram Open Network”变更为“The Open Network”，并发行了代币 TON Token。     

TON 是一个巨大的生态项目，而不仅仅是一条公链，还包括P2P网络（目前支持 Fiat 与 Crypto 之间的结算等）以及一堆服务性技术栈，比如 TON Proxy（隐藏 IP 和信息）和 TON Storage（TON 推出的分布式存储）。 

TON 目前提供的 Tech Stack，寄希望能够解决包括 Telegram 在内所有社交产品用户使用区块链或分布式服务难的问题。

而 TON Blockchain 是基于 PoS 共识机制以及 TVM 智能合约计算的公链。其理论上支持百万级上链交易，Validator 足够多的情况下，最多支持 2 ^ 92 次分片。目前，由于总交易量是有限的(当前)，目前没有验证其高 TPS 下稳定与安全性的机会，TPS 一直维持在 6 w 以下。

实现如此高 TPS 的核心特点就是 TON Blockchain 本身支持交易并行计算与验证，以及智能合约的异步处理。其架构主要分为 Masterchain、Workchain 以及 Shardchain：最大支持注册运行 2 ^ 32 条 Workchain，以及每条 Workchain 支持 2 ^ 60 次分片，及对应的 Shardchain。

可以把每个 Shardchain 理解为一个或多个 Account 的账本，由一批 Validator 进行区块构建与验证；在 Workchain 的下一个时隙，该 Workchain 下的所有 Shardchain 验证者进行再此 PoS 以获取区块奖励。所有的 Workchain 和 Shardchain 状态都会更新到 Masterchain 上。从逻辑上来说，有 5 个重要的点:

  • Shardchain 与 Workchain 都是并行计算

  • TVM 能够支持异步计算

  • 支持瞬间跨分片交流

  • 账本有多次 copy，因此对存储要求更高

  • 两次 PoS 过程才能确保 Shardchain 和 Workchain 去中心化

前三个点构成了 TON 高 TPS 的理论基础。

Interesting Connection between TG and TF

TON 目前主要由 TON Foundation 推动发展。TON Foundation 有 40 多人的团队规模，且超过一半来自于俄罗斯、乌克兰等国家地区，大部分员工都有 VK 和 TG 的工作经验。

TON FoundationTON Foundation 的一位核心创始成员 Andrew Rogozov 则是 VK.com 的前任 CEO。

TON Ecosystem Landscape

1. Validator

TON 是基于 PoS 的公链，因此验证节点规模能够确保其去中心化且稳定安全的运作。TON 目前拥有 342 个验证节点，质押量为 4.9 亿 TON，质押占比为 9.6% ，APY 能够稳定维持在 5.5% 左右。

成为 Validation Nodes 的条件如下：

  • 质押至少 300 k TON token

  • 达到硬件条件

而关于 Slash 机制，TON 仍在不断优化。当前验证者可以获得区块奖励以及用户交易费用（Gas），但如果验证者行为不当，那么他们的部分或全部股份将被削减，并销毁等量的 TON token。

2. Dev Community

数据截至 2023 年 7 月，全网全职开发人员为 6793 人，每月活跃开发者为 21338 人。而 TON 上全职开发人员为 39 人，每月活跃开发者为 174 人。

在技术发展层面，TON Foundation 在积极拓展、支持开发者社区，并通过不断完善开发者文件帮助开发者熟悉新语言以及部署并行计算环境的区块链项目（并行计算可能导致更大的开发量以及管理成本）。这次大会后有显著的关注量提升，以及一些项目开始在 TON 上启动。

3.TON’s Application

Telegram App 生态的 Web 3 部分已经向 Wallet、Game、Social、NFT、DeFi 等方向拓展，TON 生态正在加速发展，但目前主要是一些钱包和 DeFi 项目:

a. Wallet  

To C 钱包项目目前非常多，但胜在使用上都会与 TONSpace 做集成，在未来存在其他头部钱包项目比如 Metamask 入局的可能性。

Telegram Wallet (@Wallet)

Telegram Wallet 是 Telegram 原生嵌入式的中心化托管式钱包，用户目前可以在其中交换 TON、BTC、USDT。通过对话框的附件菜单栏，用户就可以和联系人发送和接收这些加密货币。

@Wallet 支持通过银行卡或 P2P 市场汇入法定货币，用户可以选择支付货币的种类，同时也支持从其他钱包存入 Cryptocurrencies。截止今年 6 月份，Telegram Wallet 注册用户超过 250 万，目前这一用户数量在飞快增长。

TON Space

TONSpace 是嵌入@Wallet 中的非托管钱包，在 Tonspace 中可以使用助记词导入现有 TON 区块链钱包或通过助记词创建新的 TON 区块链钱包，用户也可以用 Telegram 账号和邮箱的方式保存相应的私钥。当前，TONSpace 除了可以链上 Token 转账，还支持 NFT 的相关操作。

在未来，用户将可以通过 TONSpace 直接使用一些 TON 的链上设施和应用，比如与 DeFi 交互等。在 Token 2049 上该产品 Beta 版本第一次宣发，并将在今年 11 月份上线主网正式版。

b. DeFi  

在目前 DeFi 的生态不算完善，只有现货 DEX 以及 Liquid Staking，但这种开发和发展的进度在加速，不论是开发者侧、用户侧还是更多的 bridge 和 token 部署。

Megaton Finance

Megaton Finance 是一个 AMM-Based 的 Spot DEX， 韩国区块链公司 OZYS 对其进行了为期 6 个月的开发, 当前的锁定总价值 (TVL) 达到 713 万美元。

代币 MEGA 总量为 1 亿，流通量 514 万，流通市值约 71 万美金。今年 3 月完成 150 万美元种子轮融资，TONcoin.Fund 领投，Orbs、MEXC Ventures 等参投。

Bemo

Bemo 是 TON 上的流动性质押项目，与 Lido 机制类似，目前的 TVL 在$ 2.6 m 左右的水平，APY 在 5% 左右。

4. TON Token

TON Token 是 TON Blockchain 的原生代币。目前 TON Token 的供应量超过了 5 b 枚，主要是早期 TON Blockchain 的 PoW 机制供应的，在 2022 年 6 月已经全部挖完。另外，TON Token 的供应是无上限的，将以每年约 0.6% 的速度通胀增发（约 3000 万枚）。

TON Token 的 Utility 是极其差异化的。除了主流币种的投资属性、Gas 支付效用，以及 BTC 作为另类大额支付工具外，Telegram 的社交场景能够足够赋能 TON Token 作为通用支付工具，其更大的应用场景将会是多样化的社交转账、出入金、广告支付、应用支付（含 Bots）甚至商品支付，对用户来说是超过微信支付使用体验的选择。Telegram 已经在着手提供更用户友好、便捷的二级支付方式。

当前，TON 的每笔交易的平均 Gas 费用在 0.1-0.5 美金左右，远低于 Tron 的 1-2 美金，以及以太坊的 7 美金。

TON’s Comparison

TON 与 Telegram 的结合，让整个生态的天花板变的非常高，目前我们梳理出了三个比较重要的比较维度，主要是社交平台比较、区块链发展比较以及 Telegram 生态内产品形态比较：

1. Social Platform Comparison  

目前世界范围内的熟人社交平台，最主流的形态即是即时社交 (Instant-messaging)，比如 Wechat 和 WhatsApp。我们都认为这类项目有非常强的护城河，主要体现在规模效应与网络效应。随着小程序的不断发展， 22 年交易额以达数万亿，至今 MAU 已达 1.1 Billion，大量的价值转移到 APP 的内置程序中，这无一不论证这种集成式的产品形态被用户更加偏好。

基于区块链与分布式技术架构，Telegram 能够借助 Internal-Dapp 的新产品形态，发展多国家地区的实体与虚拟经济，并更大程度的降低跨区域商业的交易摩擦。如果我们将 TON 当作 TG 的商业化手段，这个故事就非常清晰。

当然，还有一个视角是 TApp 与 Web 3 Social 产品对比的视角，更加成熟的用户群，更大的代币使用场景造就 TON 链上社交产品（包括社交游戏）的落地环境要比在其他链好很多，而 Telegram 本身并不对用户数据进行分析，我认为这将吸引很多开发者和创业者的关注。

2. Blockchain Comparison  

从理论架构来说，TON 与以太坊的区别主要体现在分片数量与跨分片交流上，同时更短的 Block time、支持并行计算与异步处理也让 TON 项目部署的技术要求更加严苛，安全考量更高。         

当然好处也在于理论的 TPS 会更加高，TON 是 Ethereum 的十万倍左右；而相比于 Solana，一致的是其虚拟机都支持异步计算，区块链也支持并行计算，但明显 TON 要更加去中心化，且 Solan 并没有分片设计，所以 Solana 理论最高的 TPS（50000 ～）也低于目前 TON 的 TPS 表现。

当我们思考增长空间时，我们会发现 TON 面临了更加直观的增长点。Telegram 是世界范围内主流的社交平台（还有 VK.com 的流量）且是最大的加密群体社交平台，大量用户都对 Crypto 和 Web 3.0 存在认知，教育成本更低。这意味着 TON 的用户发展拥有相对更低的市场风险。  

而在巨型社交网络赋能下的 TON Token 将有更多的支付效用，这意味着 TON 有着更高的天花板。

3. Telegram Bots vs TON App  

Telegram Bots 是一种区块链的新交互方式，实现方式是在 Telegram 上借助 Bots 模块集成多种功能，使用户能通过向 Bots 发送指令来执行这些功能，目前主要与交易相关。这里的 Bots 项目泛指接入以太坊等非 TON 链的项目。

Bots 响应速度受限，交互方式单一且多交互并行能力差；但 TON App 能更好解决这个问题 Bots 很难为用户解决支付和出入金，但 TON 生态应用可以无缝衔接对应的基础设施

正如前文的总结，Telegram Bots 和 TON 上的项目都能够更好的直面用户，但 Telegram Bots 的交互形式更像是一个单调的代理页面，而 TON 上的项目是一个功能交互性、可组合性更强的 APP，并且上了 APP Center 的项目都可以被@Wallet(多个 TON 上的钱包集合以及 Fiat-to-Crypto)给连接起来，因此 TON 与 Telegram Bots 的项目的纵深差别非常大。

 Bots 项目的发展其实也为 TON 提供了一个更大的叙事空间，即围绕 Telegram 流量做增量的同时，也能够寻求存量加密用户的转化。因此在未来，我们不确定这些非 TON 链 Bots 是否存在被 Block 的风险，但 TON Foundation 也完全有动力向好的 Bots 项目抛出橄榄枝，尝试转化到 TON 上。

总结

面对来自技术架构、经济模型、团队等的风险，TON 目前也在积极应对：包括大规模投资社区开发者以提供更加稳定可靠的技术架构、投注社区基金以分散资金的集中度，以及与 DWF Labs 等机构计划重新设计经济模型等。

TON 的发展背靠 Telegram，我们尝试描绘未来的蓝图：提供一个链上低费用、高并行交易环境，用户能够体验各种用户友好的“部分/完全去中心化”应用，拥有清晰的链上画像与社交图谱，再到自由出入加密世界与真实世界等等，这是一个完整的网络用户生命之旅。对于未来的展望，我们不一定能够准确地想象到新的平衡是否依旧向 Web 2.0 倾斜，但能够想象到未来的网络用户会如何丝滑的使用各种信任或去信任工具以满足自身需求，Web 3.0 或许只是一种选择，但新的平衡趋势正是我们所追求的。

总体来讲，TON Ecosystem 增长点直观，生态正在加速发展，而当前生态中已部署的项目也都在很早的发展阶段，但这也创造了更多的投资机会，赔率和天花板都极其适合风投机构下注；另外，对于过早进入的新叙事，对于机构而言还有潜在的影响力收益。

但区别于其他公链，TON 的技术发展目前主要由 TON Foundation 和早期 involve 进来的开发者跟进，来自其他项目发展基础设施的机会目前存在很大的不确定性，比较难 Top-Down 的对 Infra 项目提出一些前瞻性投资建议，更适合机构的方向是尽可能多的 build connection，寻找现存问题的同时了解 TON 本身的动向。而对于 TON 生态中去中心化金融（DeFi）项目的发展，头部项目迁移至 TON 的可能性较大，由于这些项目或已较成熟，且更有潜力获得生态支持，我认为也比较难有较大的投资机会。

综合来看，基于对 TON 的发展现状，我对“加速 TON 发展”、“Internal-App 是最好产品形态”、“符合 TG 新社交范式并能创造新价值流通”的叙事一致看好，以下给出一些合理且风险更低的投资与创业方向：

Dev Tools/Community

  • Internal-App 作为更好的产品形态

  • 新价值创造与流通
