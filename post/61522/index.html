10 月 19 日，Lightning Labs 发布 Taproot Assets 协议，允许在比特币网络和闪电网络上发行稳定币和其他资产。bioniq CEO Bob Bodily 在 X 平台发文介绍详细介绍了该协议，Odaily星球日报将其编译如下：

文章精华

  • Taproot Assets 是一个与闪电网络直接集成的比特币元协议，支持 FT（同质化代币）和 NFT（非同质化代币）；

  • Taproot Assets 不使用比特币作为完整的数据可用性层（就像闪电网络）。用户默认存储自己的数据，或者可以通过离链数据存储（称为 Universe）添加额外的信任假设；

  • Taproot Assets 是一个出色的同质化代币协议，但在非同质化代币协议方面表现平凡（暂时）。

Taproot Assets 是什么

1. Taproot Assets （Taproot 资产）是一个用于 Bitcoin 的元协议，类似于 Ordinals、BRC-20、Runes、Atomicals、TAP、PIPE 或 RGB。它能处理可替代的，半可替代的和非同质化代币。

2. Taproot Assets 完全基于 UTXO（与 Runes 和 PIPE 一样），这意味着它能与 Bitcoin 原生技术（如 RGB、Lightning 和 DLC 等）很好地集成。

3. Taproot Assets 与 Lightning Network 直接集成。这意味着你可以使用 Taproot 交易来启动一个 Lightning 通道，并且可以将 BTC 和 Taproot Assets 在单个 Bitcoin 交易中存入 Lightning 通道。

4. Taproot Assets 使用 Taproot 交易在 Bitcoin 上创建资产。数据存储在 Taproot 树中，默认情况下，代币元数据在创建者的设备上存储，也可以选择存储在称为“Universe”的离链数据仓库/索引器中。

Taproot Assets 上的可替代代币

作为一个可替代代币协议，Taproot Assets 具有以下特点：

  • 低 UTXO 占用（类似于 Runes/PIPE），因为用户可以在单个交易中创建或转移多个代币；

  • 用户可以将大多数交易数据存储在链外自己或 Universe 中，这意味着大额交易的链上占用很小；

  • 它完全基于 UTXO，因此可以很好地与 Bitcoin 的 Lightning、DLCs 和 RGB 等集成；

  • 它直接与 Lightning 集成，因此在转移到 Lightning（所有代币+BTC 在单个交易中）后，用户就可以进行快速便宜的交易。

Taproot Assets 在灵活性方面具有很多潜力，Taproot Assets 表示：“个人账户持有人之间如何交互不是 Taproot Assets 规定的，但可以是特定于应用程序的。发行者在定义资产或打算限制这些资产的方式上具有灵活性。”

Taproot Assets 上的非同质化代币

然而，关于非同质化代币，Taproot Assets 与 Ordinals 或 Stamps 非常相似，用户可以使用 Bitcoin UTXO 创建和转移资产（可以是 ERC-721 或 ERC-1155 资产）。区别在于，Taproot Assets 的图像和元数据不存储在链上。理论上可以存储在链上，这意味着可能有一种方法来利用 Bitcoin 作为数据存储层，但默认情况下，图像和图像元数据都存储在链外。

对于数据有效性，Taproot Assets 具有双重效率优势。也就是说，用户可以用很小的交易创建一千个代币或空投给成千上万个地址（大部分数据都在链外，只有少量数据实际存储在链上）。

这实际上是很好的。但是，如果丢失了链外数据，用户将永远失去其代币。

这可能看起来是一个重大的限制，但这就是 Lightning Network 目前的工作方式，没有将 Bitcoin 作为完全的数据有效性层，这意味着用户要信任自己（或其他节点）来索引和保存数据，否则可能会失去其代币。

因此，Taproot Assets 不利用 Bitcoin 作为数据有效性的层 ，这与 Ordinals、 BRC-20、Runes、PIPE 等不同。

这意味着 Taproot Assets 是一个非常高效、占用链上空间小的元协议，但用户需要依靠链外的 Universe（或自己的数据存储）来保护其代币。

Taproot Assets 信任假设

由于 Taproot Assets 是一个 Bitcoin 元协议，它具有与所有其他 Bitcoin 可替代代币协议相同的信任假设，即：

  • 用户依赖链外索引器来维护状态（无论用户是否自己存储状态，对于所有其他 Bitcoin 元协议都是如此）；

  • 用户依赖自己或 Universe 来维持对其代币的拥有权。这是对 Taproot Assets 的额外假设，超过了大多数其他 Bitcoin 可互换代币协议，因为如果链外状态丢失，将失去其代币；

  • 错误的 Universe（或错误的本地状态）会导致在进行 Taproot Asset 与你的 BTC 之间的 PSBT 交换时丢失 BTC（与 BRC-20 或 Ordinals 一样）；

  • 一旦将 Taproot Assets 存入 Lightning 通道，不确定这些假设是否会改变（如果有的话），此时用户可能是继承了所有 Lightning Network 的信任假设。

Taproot Assets 如何在闪电网络上工作

对于 Lightning Network，可以使用 Taproot Assets 在 Bitcoin 上创建一个新的稳定币 taUSD（1 比 1 与 USDC 资产挂钩），然后使用单个 Bitcoin 交易将 BTC 和 taUSD 转入 Lightning Network 通道，这样用户就可以使用 taUSD 稳定币支付 Lightning 发票或进行其他 DeFi 操作。

总结一下，Taproot Assets 在 Lightning 上的原生集成使得稳定币支付成为可能，直接将 Taproot Assets 放入 Lightning 通道非常方便，它有潜力在比特币 DeFi 中发挥作用。

由于 Taproot Assets 与 Lightning 的原生集成，将其称为快速桥接到L2，因为它是内置在协议中的。用户不需要将代币桥接到侧链或主权 Rollup 上，只需要使用 Taproot 交易将 Taproot Assets 存入 Lightning 通道，然后就可以开始了。

总结

  • 我喜欢 Taproot Assets，因为它与现有的比特币可互换代币协议有不同的权衡，这使得它具有优势和劣势，更有可能解决不同的问题，吸引不同的用户，并在不同场景中发挥良好的作用。

  • 我喜欢与 Lightning Network 的直接集成。我知道 Lightning 并不是比特币的终极扩展解决方案，但是能够完成单个 Taproot 交易并快速交易 Taproot Assets 绝对是非常不错的。

  • 像往常一样，仍处于非常早期的阶段。V 0.3.0 alpha 版本意味着仍然会有一些错误。因此，如果正在测试它，请小心。
