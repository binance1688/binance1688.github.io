文章来源：Outlier Ventures；翻译：金色财经xiaozou

所有技术都在快速发展，Web3领域的表现尤为显著。因此，我们通过Web3 Tech Radar提供了一个不断更新的发展状态图。

创新技术的发展——其中大部分是开源的——意味着新的开发人员通常可以利用工具和平台更快地与底层区块链交互，而不必从头开始编写集成。

即使一个人只专注于一个单一的生态系统，比如以太坊及其他EVM兼容网络，也会有太多的情况让人目不暇接，以至于很容易错过新的平台、工具，甚至是让生活更轻松的用例。

1、Web3 Tech Radar简介

Web3 Tech Radar来源于ThoughtWorks开创的Tech Radar（技术雷达）理念，Tech Radar是“对技术前沿有态度的趋势报告”，我们将每六个月左右对基础设施、语言和库、工具和主题生成一次快照，这些都属于当前的Web3技术领域范畴。每一项都被划分为采用、试用、评估和保留四个类别中的一类。

  • 采用：发展成熟。

  • 试用：日益成熟，适合你的业务范围。

  • 评估：值得探索和/或运行概念验证。

  • 保留：有更好的替代方案。

一个典型的生命周期可能会看到一个元素首先被评估，然后被试用，然后被采用，直到被更好的选项取代。

在Outlier Ventures，我们决定利用我们丰富的知识，亲自试验这些技术，看看我们的Web3生态系统中的250 多家投资组合公司是如何使用这些技术的。我们的产品和工程团队成员密切合作，支持和指导我们的加速器项目。我们采用数据导向的方法，记录了去年约200 次技术尽职调查结果，这为我们提供了对Web3工具、趋势、库和基础设施的现状的独特见解。

2、Web3 Tech Radar重点内容

让我们深入了解一下第一版的Web3 Tech Radar，具体来看四个方面：基础设施、语言和Lib、工具和主题。

（1）基础设施

有几个项目已经在基础设施层实现了相当大的改进。运行节点、调试交易或部署dApp比以前要更加简单。我们注意到Paradigm通过赞助以及对技术栈的不同层做出实质性贡献来展示其实力。Reth是他们在基础设施层上带来的，使用Rust语言实现了一个新的以太坊全节点。

L2空间对开发者的吸引力在增长，得到开发者越来越多的采用。到目前为止，Optimism Stack和Arbitrum Orbit是两个获得更多关注的平台，但ZK解决方案（zkSync、Scroll、Polygon zkEVM ）正在加紧努力，我们可以期待在未来几个月里，rollup生态系统将受益于Proto-Danksharding (EIP-4844)带来的影响，届时会有更多进展。大多数探讨都围绕着如何凭借EVM兼容性或EVM等效性来吸引开发人员。

（2）语言和Lib

关于将web3钱包关联到dApp的战斗正在进行中。感谢WalletConnect、Web3-Onboard和Web3-React等库，它们让这种关联变得越来越容易。与此同时，我们看到围绕着EIP-6963正在展开一些行动，以改善用户体验。

在前端层，类似于在基础设施层所发生的，Paradigm正通过支持诸如Wagmi、Viem和Rivet等库的开发而发挥作用。

同时，我们看到Rust扩展到了基础设施层之外，甚至又通过Arbitrium Stylus alpha版本编写EVM兼容智能合约而回归。

在ZK领域，出现了Circom、Noir和Leo等语言来帮助开发者编写电路；安全审计员正在提升相关技能。

（3）工具

开发者工具正在匀速改进，VSCode、Hardhat、Slither和Mythril等工具可支持智能合约开发者的核心需求。随着Foundry和Ether的吸引力上升，Rust的使用率也见涨。Rust正在成为那些寻求更快开发体验的开发人员的常用工具，但它还没有完全发展成熟，不会很快取代Javascript/Typescript。

另一个验证生态系统成熟程度的有趣模式是以太坊基础设施开发公司 Consensys 关闭其开发工具 Truffle 和 Ganache，并与 HardHat 建立新的合作伙伴关系。最终，维持开发人员框架的最新状态是真正要做的工作，而在同一个解决方案背后拥有多个实体是有意义的。

（4）主题

过去的几个月里，在播客和博客文章中，我们看到了许多关于模块化区块链、rollup 即服务（RaaS ）以及如何最终实现可扩展性的讨论。如何实现这些还有待讨论，但最有趣的还要属模块化区块链，开发人员可以自定义和微调所有细节，或者放手采用RaaS方式而无需或只需要很少的基础设施知识。

在我们的ZK-1项目中，我们与Radius达成合作，Radius是模块化区块链的共享排序层，我们有机会观察该领域是如何从概念发展为真正有用的产品的。

另一个主要的促成因素是正在成为现实的帐户抽象。今年5月，ERC-4337并入上海以太坊分叉后，在所有EVM链上刮起了一股UX创新之风。目前，很多供应商——例如我们的两家投资组合公司Bionomy和Etherspot，以及Thirdweb、Alchemy、Safe等——都致力于提供精简优化的用户和开发者体验。我们可能会在未来看到整合，但就目前而言，我们看到的是一种健康趋势。

3、总结

Web3 Tech Radar的EVM版本展示了开发智能合约和dApp变得越来越快、越来越有趣；感谢前端和后端技术成熟的生态系统。但最需要关注的还是安全方面的改进。程序化货币是一项了不起的功能，但需要适当的安全措施才能充分发挥其潜力。

一些新的安全工具正日渐成熟（例如Scribbles、Forta和Ziion），但我们距实现默认可信的程度还有很长的路要走。

尽管做了这样那样的尝试和努力，人工安全审计仍然是预防漏洞攻击最重要的防御手段，但由于所需的财务资源和时间，人工安全审计也限制了链上开发和创新的速度。

另一个重要进展是L2现在已是真实存在的。它们获得了一定关注度，Optimism Stack和Arbitrum Orbit这两个平台是舞台的中心，但ZK解决方案（zkSync、zkScroll、Polygon zkEVM）正在加紧努力，我们可以期待在未来几个月里，rollup生态系统将受益于Proto-Danksharding (EIP-4844)带来的影响，届时会有更多进展。

我们将定期（约6个月）迭代推出Web3 Tech Radar，提供生态系统发展状态的更新快照。
